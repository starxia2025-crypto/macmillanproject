Quiero que generes una aplicación web moderna, responsive, multiusuario y multicliente, lista para producción, que sustituya completamente el sistema actual basado en SharePoint + Microsoft Forms + Microsoft Lists + Power Automate.

## Objetivo general
La aplicación debe centralizar en una sola plataforma web:
- la gestión documental por cliente (manuales, tutoriales, vídeos, ayuda)
- la creación y seguimiento de incidencias/consultas
- las notificaciones automáticas por email
- la administración de usuarios, roles y permisos
- panel técnico interno
- estadísticas, informes y auditoría

La aplicación debe funcionar con una sola base de datos PostgreSQL alojada en mi VPS, usando arquitectura multitenant con:
- una misma base de datos
- un schema separado por cada cliente
- aislamiento claro entre clientes
- posibilidad de escalar a nuevos clientes fácilmente

## Requisitos de arquitectura multicliente
Implementar multitenancy real con estas reglas:
- una sola base de datos PostgreSQL
- un schema distinto por cliente
- los usuarios de un cliente solo pueden ver datos de su propio schema
- los técnicos internos pueden acceder a varios clientes según permisos
- debe existir un tenant manager o capa de resolución de tenant
- el tenant se debe resolver por dominio, subdominio, slug o configuración del usuario autenticado
- toda consulta debe ir siempre filtrada y aislada por tenant/schema
- evitar fugas de datos entre clientes

## Objetivo funcional
La app debe permitir:
- que cada cliente tenga su portal propio
- mostrar documentación, ayuda, vídeos y recursos
- crear tickets/incidencias de forma fácil y amigable
- adjuntar archivos si es viable
- hacer seguimiento del estado del ticket
- enviar notificaciones por email al equipo técnico
- enviar confirmación al usuario que creó el ticket
- enviar notificación automática cuando el ticket se resuelva
- permitir respuestas internas y externas
- tener trazabilidad completa
- disponer de dashboard y estadísticas
- exportar informes a PDF y Excel
- enviar informes por email
- registrar auditoría y cambios en base de datos

## Contexto actual a sustituir
Actualmente cada cliente tiene su propio SharePoint donde:
- se muestran manuales, tutoriales, vídeos y contenido de ayuda
- los usuarios suben incidencias mediante Microsoft Forms
- los datos se guardan en Microsoft Lists
- Power Automate envía correos al equipo técnico
Quiero eliminar toda esta dependencia y concentrarlo en una única aplicación web profesional.

La seguridad es prioritaria. Implementa al menos:
- autenticación segura
- autorización por roles y permisos granulares
- sesiones seguras
- aislamiento estricto por tenant
- validación estricta de inputs
- consultas parametrizadas

## Roles y permisos
Implementar sistema RBAC flexible, con roles por defecto y permisos ampliables.

### Roles mínimos
1. superadmin
- administra toda la plataforma
- crea clientes/tenants
- configura schemas
- gestiona técnicos globales
- ve estadísticas globales
- accede a auditoría global

2. admin_cliente
- administra usuarios de su cliente
- gestiona documentación de su portal
- ve todos los tickets de su cliente
- exporta informes
- configura notificaciones de su cliente

3. tecnico
- acceso al panel técnico
- ve tickets asignados o visibles según permisos
- cambia estados
- responde tickets
- se asigna o es asignado a tickets
- ve trazabilidad
- genera informes

4. usuario_cliente
- crea tickets
- consulta el estado de sus tickets
- puede responder o ampliar información
- accede a documentación y recursos permitidos

5. visor_cliente
- solo lectura sobre documentación o tickets permitidos

Además, quiero permisos granulares configurables, no solo roles fijos.
